docs [with-ai] FT-013 Family 관리 시스템 기획서 수정 - 무한 스크롤 적용

- 공개 Family 검색 기능을 기존 페이징에서 무한 스크롤로 변경
- 모바일 환경에서 더 자연스러운 UX 제공을 위한 기획 수정
- API 응답 구조 및 구현 가이드라인 업데이트

## 수정된 주요 내용

### API 설계 변경
- GET /api/families/public: 페이징 → 커서 기반 무한 스크롤로 변경
- 응답 구조: page 객체 → pagination 객체 (nextCursor, hasNext 포함)
- 내 소속 Family 목록: 전체 조회 (페이징 없음, 일반적으로 15개 이하)

### 구현 요구사항 추가
- CursorPageable: 커서 기반 페이징 객체
- CursorPage: 커서 페이징 응답 객체  
- CursorUtils: Base64 인코딩/디코딩 유틸리티

### 성능 목표 조정
- 공개 Family 검색 응답 시간: 1000ms → 800ms (무한 스크롤 최적화)

## 영향도 분석
- Story-017 예상 공수: 2일 → 2-3일 (무한 스크롤 구현 시간 추가)
- 전체 개발 일정: 변경 없음 (큰 변경사항 아님)
- UX 개선: 모바일 친화적 인터페이스로 사용자 경험 향상
