refactor [with-ai] PM-021 ApiResponse 래퍼 클래스 제거 - ResponseEntity 통일

- 불필요한 ApiResponse 래퍼 클래스 완전 제거
- 모든 컨트롤러를 ResponseEntity로 통일하여 Spring Boot 표준 패턴 적용
- JSON 직렬화 혼란 요소 제거로 개발 생산성 향상

## 구현된 주요 컴포넌트

### 프레젠테이션 계층
- ModifyFamilyMemberRoleController: ApiResponse → ResponseEntity 변경
- ModifyFamilyMemberStatusController: ApiResponse → ResponseEntity 변경
- 기타 모든 컨트롤러: 이미 ResponseEntity 사용 중 (변경 없음)

### 제거된 컴포넌트
- ApiResponse 클래스: 완전 삭제
- @JsonInclude(NON_NULL) 설정: 제거됨
- 불필요한 래퍼 구조: 정리 완료

### 특화 사항
- Spring Boot 표준 패턴 준수: ResponseEntity<T> 통일
- JSON 응답 구조 단순화: 직접적인 DTO 반환
- Optional 타입 JSON 직렬화 명확성: 혼란 요소 제거
- 코드 가독성 및 유지보수성 향상

## 테스트 구현
- 전체 테스트 통과 확인 (BUILD SUCCESSFUL) ✅
- 기존 테스트 코드 영향 없음 확인
- 컨트롤러 테스트 정상 동작 확인

## 해결된 이슈
- JSON 직렬화 래퍼 혼란 문제 해결
- 불필요한 추상화 계층 제거
- Optional 타입 처리 가이드라인 명확화
- 개발자 AI의 응답 포맷 혼란 감소

## 워크플로우 개선
- Spring Boot 표준 패턴 완전 적용
- 프로젝트 코드 일관성 향상
- JSON API 응답 구조 단순화
