refactor [with-ai] FT-005 메서드명 수정 - isHasRelationship → hasRelationship

- 부자연스러운 isHasRelationship() 메서드명을 hasRelationship()로 수정
- Lombok @Getter 제거하고 명시적 getter 메서드 구현
- 모든 테스트 케이스에서 메서드 호출 업데이트

## 변경 사항

### 메서드명 정규화
- BEFORE: `isHasRelationship()` (문법적으로 부자연스러움)
- AFTER: `hasRelationship()` (자연스러운 boolean 메서드명)

### Lombok 의존성 제거
- @Getter 어노테이션 제거
- 필요한 getter 메서드들 명시적 구현
- 메서드명 제어권 확보

### 코드 품질 향상
- 메서드명 가독성 개선
- 일관된 네이밍 컨벤션 적용
- 테스트 코드 가독성 향상

## 영향도
- API 호출 방식 변경: response.isHasRelationship() → response.hasRelationship()
- 모든 테스트 케이스 업데이트 완료
- 기능적 변경 없음 (순수 리팩토링)
