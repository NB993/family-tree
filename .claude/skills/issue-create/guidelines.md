# 이슈 생성 가이드라인

## 1. 이슈 타입과 라벨 매핑

| 작업 유형 | 이슈 타입 라벨 | 설명 |
|----------|---------------|------|
| 버그 수정 | `bug` | 기존 기능의 오류 수정 |
| 신규 기능 | `enhancement` | 새로운 기능 추가 |
| 사용자 스토리 | `story` | 사용자 관점의 기능 요구사항 |
| 기획/설계 | `planning` | 기획 또는 설계 관련 작업 |
| 문서화 | `documentation` | 문서 작성 또는 수정 |
| 디자인 | `design` | UI/UX 디자인 관련 |

## 2. 작업 영역 라벨

| 영역 | 라벨 | 설명 |
|-----|------|------|
| 프론트엔드 | `frontend` | React, CSS, UI 관련 작업 |
| 백엔드 | `backend` | Spring Boot, API 관련 작업 |

## 4. 실행 명령어

### 이슈 생성

```bash
gh issue create --title "이슈 제목" --body "$(cat <<'EOF'
이슈 본문
EOF
)"
```

### 라벨 추가

```bash
gh issue edit [이슈번호] --add-label "bug,frontend"
```

### 프로젝트 연결

```bash
gh project item-add 3 --owner NB993 --url https://github.com/NB993/olhaeolhae/issues/[이슈번호]
```

### 마일스톤 지정

```bash
# 마일스톤 목록 조회
gh api repos/NB993/olhaeolhae/milestones --jq '.[] | "\(.number)\t\(.title)"'

# 마일스톤 지정
gh issue edit [이슈번호] --milestone "마일스톤 제목"
```
