# 리팩토링 지침

## 목적

- 테스트 통과 상태 유지하면서 코드 개선
- 중복 제거 및 가독성 향상
- 기존 명명 규칙/아키텍처 패턴 준수

## 사용 시점

- TDD Green 단계 완료 후
- 코드 리뷰 피드백 반영 시
- 기술 부채 해소 시

## 핵심 규칙

### 리팩토링 전 체크리스트
- [ ] 모든 테스트가 통과하는가?
- [ ] 리팩토링 범위가 명확한가?
- [ ] 동작 변경 없이 구조만 개선하는가?

### 리팩토링 후 체크리스트
- [ ] 모든 테스트가 여전히 통과하는가?
- [ ] 명명 규칙을 준수하는가?
- [ ] 코드 품질이 향상되었는가?
