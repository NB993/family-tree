# PR 생성 가이드라인

## PR 제목 형식

```
{타입}: {간결한 설명}
```

### 타입
- `feat`: 새 기능
- `fix`: 버그 수정
- `refactor`: 리팩토링
- `docs`: 문서
- `test`: 테스트
- `chore`: 기타 (빌드, 설정 등)

## PR 본문 템플릿

```markdown
## 변경 목적
-

## 변경 내용

### 주요 변경사항
-

### 구현된 컴포넌트 (해당시)
-

## 관련 문서
- PRD: <!-- PRD 파일 경로 또는 없으면 삭제 -->
```

## 금지사항

- `git push --force` 사용 금지
- main/master 브랜치에서 직접 작업 금지
- 테스트 실패 상태에서 PR 생성 금지
- 1000줄 이상의 큰 PR 지양

## PR 생성 전 체크리스트

- [ ] 테스트 통과
- [ ] 현재 브랜치가 main/master가 아님
- [ ] 원격에 브랜치 push 완료
- [ ] PR 제목 형식 준수
